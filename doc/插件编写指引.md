# 插件编写指引

本文档面向当前仓库内的 AITeams 插件系统，基于现有实现整理，目标是让一个新插件能够被校验、安装、加载、调用、热重载。

相关实现入口：

- `aiteams/plugins/manifest.py`
- `aiteams/plugins/worker.py`
- `aiteams/plugins/manager.py`
- `aiteams/plugins/tool_adapter.py`
- `examples/demo_cmd_echo_plugin`

## 1. 插件包最小目录结构

一个插件包本质上是一个目录，目录根下至少要有 `plugin.yaml`，并且 `entrypoint` 指向的 Python 模块必须存在。

```text
my_plugin/
  plugin.yaml
  runtime.py
  README.md
```

最小可运行版本通常只需要：

- `plugin.yaml`
- `runtime.py`

## 2. plugin.yaml 规范

`plugin.yaml` 是插件包入口。当前系统会在 `aiteams/plugins/manifest.py` 中进行校验和归一化。

### 必填字段

- `key`
  插件唯一标识，建议使用稳定 slug，例如 `demo.hello_echo`。
- `name`
  展示名称。
- `version`
  版本号，例如 `v1`。
- `entrypoint`
  入口对象，格式必须是 `module.path:object`，例如 `runtime:create_plugin`。

### 常用可选字段

- `plugin_type`
  默认为 `toolset`。
- `description`
  插件简介。
- `workbench_key`
  工作台标识，不填时默认等于 `key`。
- `tools`
  工具分组标签，用于描述该插件暴露的工具能力。
- `permissions`
  权限标签，用于审计、风控、审批展示。
- `actions`
  插件可执行动作列表，决定模型能调用哪些动作。
- `config_schema`
  预留给配置页面使用的结构定义。
- `hot_reload`
  是否启用热重载，默认 `true`。
- `runtime`
  运行时默认配置，会在插件加载时合并到 `manifest["runtime"]`。
- `requirements`
  依赖声明，仅做描述用途，当前系统不会自动安装。

## 3. actions 如何定义

`actions` 是插件的核心。每个 action 对应一次 `invoke(action, payload, context)` 调用。

推荐写成对象，而不是只写字符串，因为对象可以声明工具名和入参结构。

```yaml
actions:
  - name: echo
    tool_name: hello_echo
    description: Echo user text and selected runtime context.
    input_schema:
      type: object
      properties:
        text:
          type: string
          description: Text to echo.
        include_context:
          type: boolean
          description: Whether to append a short context summary.
          default: false
      required:
        - text
```

### 字段说明

- `name`
  插件内部动作名，`invoke()` 里用它做分发。
- `tool_name`
  暴露给模型的工具名。
- `description`
  工具说明。
- `input_schema`
  JSON Schema 风格的对象定义。
- `output_schema`
  可选，当前主要用于描述，不影响运行。

### 为什么建议写 input_schema

`aiteams/plugins/tool_adapter.py` 会根据 `input_schema` 自动构造成结构化工具。

如果没有 `input_schema`：

- 模型端会退回到一个只有 `payload_json` 字符串参数的 legacy 工具
- 可用性和可控性都会差很多

所以新插件建议始终提供 `input_schema`。

## 4. 入口对象怎么写

`entrypoint` 可以指向：

- 一个类
- 一个工厂函数

加载逻辑在 `aiteams/plugins/worker.py::_load_runtime()` 中。系统会传入两个关键参数：

- `manifest`
- `root_path`

推荐签名：

```python
class MyPlugin:
    def __init__(self, *, manifest: dict[str, Any], root_path: str):
        ...
```

或者：

```python
def create_plugin(*, manifest: dict[str, Any], root_path: str):
    return MyPlugin(manifest=manifest, root_path=root_path)
```

## 5. 运行时对象建议实现的方法

插件运行时对象不要求继承固定基类，但建议实现下面四个方法：

### `describe() -> dict`

返回插件描述信息。通常用于插件详情和调试。

### `health() -> dict`

返回健康检查结果。

### `invoke(action: str, payload: dict, context: dict) -> dict`

真正的业务执行入口。必须返回可序列化的字典。

### `shutdown() -> dict`

插件卸载或关闭时调用。

如果某个方法未实现，`aiteams/plugins/worker.py` 会回退到 `DefaultPluginRuntime` 的默认实现。

## 6. context 中通常会带什么

插件调用时的 `context` 由 `aiteams/agent/kernel.py::_plugin_invocation_context()` 组装。常见字段有：

- `workspace_id`
- `project_id`
- `run_id`
- `team_id`
- `node_id`
- `agent_id`
- `visible_outputs`
- `available_plugins`
- `knowledge_bases`
- `memory_runtime`
- `phase`
- `message_type`
- `source_actor_id`
- `target_agent_id`
- `adjacent_targets`
- `message`

其中 `message` 通常包含：

- `message_id`
- `source_actor_id`
- `target_agent_id`
- `message_type`
- `phase`
- `body`

建议：

- 只读取自己真正需要的字段
- 不要假设所有字段都一定存在
- 使用 `dict.get()` 做兼容读取

## 7. runtime 配置如何注入

插件加载时，最终给插件看到的 `manifest["runtime"]` 是这几层合并后的结果：

1. `plugin.yaml` 中的 `runtime`
2. 外部注入的运行时配置 `AITEAMS_PLUGIN_RUNTIME_CONFIG`
3. 外部注入的运行时密钥 `AITEAMS_PLUGIN_RUNTIME_SECRET`

因此在插件代码里，推荐统一从这里读取配置：

```python
runtime = dict(manifest.get("runtime") or {})
```

## 8. 热重载机制

热重载由 `aiteams/plugins/sandbox.py::_maybe_hot_reload()` 控制。

当 `plugin.yaml` 中：

```yaml
hot_reload: true
```

时，系统会基于插件目录指纹检测文件变化并重启插件进程。

建议：

- 调试阶段保持 `hot_reload: true`
- 线上稳定版也可以保留开启
- 如果插件内部维护了外部连接，记得在 `shutdown()` 里做清理

## 9. 安装与加载流程

当前流程大致如下：

1. 校验插件目录
2. 解析 `plugin.yaml`
3. 拷贝到安装目录
4. 启动沙箱进程
5. 通过 JSON 行协议调用 `describe / health / invoke / shutdown`

默认安装目录来自 `PluginManager.install_root`，当前应用里通常是：

```text
data/plugins/<plugin-key>/<version>
```

## 10. 调试建议

### 推荐做法

- `invoke()` 只返回业务结果字典
- 用 `print()` 或日志输出调试信息
- 将所有外部依赖异常转成明确错误信息抛出
- 保持 action 名稳定，不要随意改动

### 注意事项

- `invoke()` 返回值必须是字典
- `entrypoint` 指向的模块必须在插件根目录下能导入
- `requirements` 当前不会自动安装，需要你自己保证运行环境已有依赖
- 没有 `input_schema` 时，模型只能拿到 `payload_json` 风格工具
- 插件里的标准输出会被沙箱接管，调试日志应视为 stderr 侧调试信息，而不是业务返回值

## 11. 建议的最小模板

```text
my_plugin/
  plugin.yaml
  runtime.py
  README.md
```

```yaml
key: demo.my_plugin
name: Demo My Plugin
version: v1
plugin_type: toolset
description: Minimal demo plugin.
entrypoint: runtime:create_plugin
tools:
  - my_plugin
permissions:
  - readonly
actions:
  - name: run
    tool_name: my_plugin_run
    description: Run the demo action.
    input_schema:
      type: object
      properties:
        text:
          type: string
          description: Text input.
      required:
        - text
hot_reload: true
runtime:
  greeting: hello
```

```python
from __future__ import annotations

from typing import Any


class MyPlugin:
    def __init__(self, *, manifest: dict[str, Any], root_path: str):
        self.manifest = manifest
        self.root_path = root_path
        self.runtime = dict(manifest.get("runtime") or {})

    def describe(self) -> dict[str, Any]:
        return {
            "key": self.manifest["key"],
            "name": self.manifest["name"],
            "version": self.manifest["version"],
            "actions": list(self.manifest.get("actions") or []),
        }

    def health(self) -> dict[str, Any]:
        return {"status": "ok"}

    def invoke(self, action: str, payload: dict[str, Any], context: dict[str, Any]) -> dict[str, Any]:
        if action != "run":
            raise ValueError(f"Unsupported action `{action}`.")
        return {
            "ok": True,
            "action": action,
            "echo": str(payload.get("text") or ""),
            "agent_id": context.get("agent_id"),
            "greeting": self.runtime.get("greeting", "hello"),
        }

    def shutdown(self) -> dict[str, Any]:
        return {"status": "stopping"}


def create_plugin(*, manifest: dict[str, Any], root_path: str) -> MyPlugin:
    return MyPlugin(manifest=manifest, root_path=root_path)
```
